Hello there, today we're going to go over a guide for flashing a NEO Coolcam Siren with Temperature and Humidity Multisensor using Tasmota for a local network first setup. 

This is a condensed guide on flashing with tasmota, so if you need more information, I recommend checking out the official documentation.

First things first, you'll need to solder onto the PCB, which means you'll have to open up the back plate and remove three screws. Now, this can be a bit tricky, as it requires connecting all of the points on the board. To do this, you'll need a Serial-to-USB Adapter, which will allow you to connect your device to your computer.

Let me walk you through what needs to be connected:

| **Device - Neo Coolcam** | **Flasher** |
| ------------------------ | ----------- |
| GPIO0                    | GND         |
| GND                      | GND         |
| TX                       | RX          |
| RX                       | TX          |
| 3.3v                     | 3.3v        |

If you're not familiar with these terms, don't worry, just remember that you need to connect GPIO0 to ground in order for the device to boot into flash mode.

You can find more information on this process by checking out the blog post on our website, which includes some great images to help guide you through it. 

Once you've got your connections set up, you'll need to install esptool using pip. This is a Python package that will allow you to flash the firmware onto your device.

Here's how you do it:

`pip install esptool`

Next, change the port in your terminal and run this command:

`esptool --port COM3 write_flash -fs 1MB -fm dout 0x0 tasmota-sensors.bin`

Now that we've got our device flashed with Tasmota, let's talk about configuration. To do this, you'll need to connect to the access point and set up your default WiFi connection.

You can also use a GUI version called Tasmotizer if you prefer a more visual interface.

Once you're connected, apply the template for the NEO Coolcam Multisensor. This will give you all of the settings you need for temperature, humidity, and alarm functionality.

Here's an example of what you might enter:

`{"NAME":"Neo Siren 3in1","GPIO":[255,107,255,108,255,255,0,0,255,255,255,255,255],"FLAG":0,"BASE":54}`

Now that we've got our device configured, let's talk about some of the Tasmota console commands you can use. These will allow you to control your device remotely and get more out of its functionality.

| **Command**                                       | **Reason**                                         |
| ------------------------------------------------- | -------------------------------------------------- |
| `Backlog SetOption65 1; SetOption66 1; Module 54` | Fast power cycle and register as a Tuya module     |
| `TuyaMCU 11,104`                                  | Map alarm siren/led to toggle switch               |

| `TuyaSend2 113,0`                                 | Disable temperature alarm. Do with a rule instead  |

Keep in mind that all of this assumes you have MQTT configured on your network.

Once you've got everything set up, the data from your device will be polled and reported to your dashboard every so often.

You can also use NodeRed to catch the Tuya event and publish the MQTT message in that rule. This allows for a lot more flexibility when it comes to setting up smart logic on your network.

Here's an example of how you might set this up:

```javascript
msg.measurement = 'siren-tv';
try {
    var payload = JSON.parse(msg.payload);
    if(payload.TuyaSNS){
        msg.payload = {};
        if(payload.TuyaSNS.Temperature){
            msg.payload.Temperature = payload.TuyaSNS.Temperature;
        }
        if(payload.TuyaSNS.Humidity){
            msg.payload.Humidity = payload.TuyaSNS.Humidity * 10;
        }
    }
} catch(ex) {
    node.error("unable to get temp and humidity", msg);
    return;
}
return msg;
```

This will allow you to turn on the siren remotely by sending an ON or OFF message to the MQTT topic.
