You're probably familiar with the problem of leaving your garage door open, and I'm no exception. In fact, I've left mine open too many times. If you're looking for a solution to automate your garage door, you can find this tutorial on my blog.

The goal of this project is to use a Raspberry Pi to control your garage door and some other features, like turning the light on or off. We'll be using Docker to make it easy to share and restore the setup.

To get started, you'll need a few components. You can find them listed in my blog post. The main ones are relay modules, a replacement switch for your garage door opener, and a Raspberry Pi 3. If you want to add a camera to monitor what's happening near your garage, I recommend using the Sony Play Station Eye Camera.

As I was researching this project, I came across many other people who have created similar setups. However, my garage door system has some anti-tamper hardware measures that made things more complicated. But with a little creativity and some trial-and-error, I was able to find a solution.

One of the key parts of the setup is the wiring. You'll need to connect the relay module to the Raspberry Pi and the LiftMaster circuit. The image shows what the setup looks like when it's all connected.

Once you have everything wired up, you can install Docker on your Raspberry Pi. I've included a link to a tutorial on my blog that explains how to do this.

After installing Docker, you'll need to clone the repository for this project and run the build command to create an image. Then, start the container in privileged mode and map port 8090 to allow access to the web API.

The web API provides several endpoints that you can use to control your garage door and light switch. You can also check the status of the garage door by sending a request to the /api/garage/status endpoint.

If you're interested in learning more about this project, including the code and setup details, be sure to check out my blog post on the topic.

This project took me about two days to complete, with some testing and debugging along the way. I'm considering moving the GPIO logic to Python and using Node.js as the API router. If you have any suggestions or would like to contribute to this project, feel free to submit a pull request!
