Part Two of Hacking My Sony A6000 Camera

This is a smaller continuation of the previous blog post on hacking the Sony A6000 camera. Come back for more updates.

Let’s talk about free apps. To install time-lapse and an FTP uploader with some additional hacks, an open source project was awesome. You can find the link on the blog post. Download the executable from the releases, then it will provide you with the applications to install. I would also recommend the OpenMemories Tweaks app as it allows changing some limits on the camera.

Next, the USB-C battery hack. Batteries drain very quickly with this camera. The stock battery is around 1020 milliamphours. Swapping out batteries is common and tedious. Getting USB-C versions of them helps with charging on the go, but it’s still annoying with cables.

After a bit of hacking the power adapter version of this, I bundled everything up into a single battery shell with a USB-C female tail. Nothing clever here other than taking apart the USB five-volt to eight-volt adapter, then adding that to the shell with a USB-C input. The idea here is to use a large battery bank when taking very long time-lapses. Building a battery pack on the bottom is the next idea. Here is the build for the power adapter. A video is shown here demonstrating the build process.

Now, for data transfer hurdles, a follow-up from my previous post.

First, using a card reader. On the go, I keep this card reader near the camera bag. It allows for the SD card to be plugged into the computer or phone in a small package. Sometimes I just need a few pictures on my phone to edit and send. All the other files get dumped.

Second, using the FTP uploader within the camera. This idea of using the FTP uploader is great, but it’s very slow. Seventy files in thirteen minutes at full resolution JPEG files. If there aren’t a lot of pictures, this makes sense, but typically taking the SD card out and data dumping is going to be much quicker still.

Using the FTP uploader app mentioned above, I use this bit of Python code to dump it into the directory that I care about. The command runs a Python module to start an FTP server with a user and password set for write access. There is a reference to this command on the blog post.
