Save JSON Objects to AWS S3 with Google Recaptcha

This post will show you how to save messages from a public endpoint to your backend using AWS Lambda, AWS S3 and Google Recaptcha.

I wrote the gist for myself, but I had enough requests to share it in a blog post. Feel free to copy and paste what you want.

Request from a frontend source

Add this to the frontend index.html, or anywhere a form will be sent from. Replace the underscore recaptcha site key with a real one.

First, include the Google Recaptcha script source. Then, within a script tag, use grecaptcha ready to execute the site key with the action set to homepage, and handle the resulting token.

You can find the official documentation for Google Recaptcha version 3 on the Google developers website.

Backend Setup

Start by creating an AWS Lambda function that has access to add items to S3. Use a node.js javascript lambda function.

You will need to set up the following environment keys:

First, GoogleRecapKey. Get this from the Google Recaptcha developers page.

Second, MessageBucket. Create a bucket in AWS S3.

Third, SnsTopicArn. Create a SNS topic to receive emails or SMS to your phone.

For the code implementation, you can find the link to the gist on the blog post.

API Gateway Setup

To set up the API Gateway, follow these steps:

First, create an AWS API Gateway API with a resource named form with a method of POST.

Second, set the integration type to Lambda Function.

Third, ensure that Use Lambda Proxy integration is checked.

Fourth, select your Lambda function.
