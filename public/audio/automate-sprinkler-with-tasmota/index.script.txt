You're probably tired of using those over-the-shelf sprinkler systems that are difficult to use and lack customization options. They may also be outdated and not as efficient with water or power usage as you'd like. The "set it and forget" mentality isn't always the best approach, is it? It might be worth considering a more customizable and efficient option for your home's sprinkler system.

Let's talk about improving your home's sprinkler system without breaking the bank. One solution is to use a combination of inexpensive hardware and customizable open-source software. This approach gives you flexibility and the ability to add additional features as needed. The main goal here is to trigger the water sprinkler solenoid to turn on or off based on specific conditions such as time, including sunrise and sunset, humidity, and temperature. By implementing this method, you can achieve better results with low effort.

Now, let's talk about what we need for our project. We're looking for an IoT device that runs when the internet is down, timer-based watering with support for sunrise and sunset, manual toggling of the watering state, backup and restore settings... and a few other features we'd like to have.

For example, it would be great if we could measure water time, humidity, and temperature. We'd also like local network control without submitting data to a third party, graphing and analyzing the data, watering runaway protection, and more.

At the lowest level, our IoT hardware should do four things: toggle power on or off, report power state, humidity, and temperature, and not require an internet connection. Now, I know there are some over-the-shelf devices that can do most of these things easily, but we'll need to add some special programming to get the full functionality.

One option is using a Sonoff device, which can be found on Amazon. It's under $20 USD and does three out of four things easily. We can wire this up by cutting into an existing extension cord and following the directions in the Sonoff guide. Plug one into the 24 VAC power supply of your existing water sprinkler adapter.

I ended up using alternative hardware that includes an Esp8266 microcontroller, OLED screen, solid state relay, and a BME280 for temperature, humidity, and pressure measurements. Now, let's talk about software. We can flash our device with open-source software called Tasmota, which solves several problems: local network control, running even when the internet is down, high customizability, simple programming... and more.

Using the Tasmota Web Console Command line, we can set up our timers and parameters. For example, we can use `PowerOnState 0` to default the power state to off if there's a power outage or reset. We can also set our time zone using a standard GMT offset: `Timezone -8`. This doesn't account for daylight savings time, so be sure to check the Tasmota docs.

Next, we'll create four timers in the Tasmota Timer Web UI. Two of these will start and stop the watering at sunrise, while the other two do the same at sunset. We can use `Timer1` to enable starting the watering at 00:00 with a 20-minute offset. The `Timer3` will stop the watering after the 20 minutes are up.

We'll also want to add some runaway protection to our system. We can use Timer2 and Timer4 to stop the sprinkler, never run more than 1,320 seconds (22 minutes) when the power state is on... or we can even use NodeRed to trigger a power state off if receiving MQTT messages.

Finally, let's talk about extending this project to other platforms. Using a simple flow in NodeRed, I've added support for a UI and Alexa control with tracking usage via Graphana. We can use the MQTT message `tele/tasmota_YOURDEVICE/SENSOR` to format and send data to our local Graphana instance on a Raspberry PI.
