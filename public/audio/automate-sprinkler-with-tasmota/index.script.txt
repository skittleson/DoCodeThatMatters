A simple water sprinkler system, for makers!

Homeowners often find that over-the-shelf sprinkler systems can be difficult to use and lack customization options. They may also be outdated and may not be as efficient for water or power usage as desired. The set it and forget mentality is not always the best approach. It may be worth considering more customizable and efficient options for your home's sprinkler system.

Project Research

Want to improve your home's sprinkler system without breaking the bank? One solution is to use a combination of inexpensive hardware and customizable open-source software. This approach allows for flexibility and the ability to add additional features as needed. The main goal is to trigger the water sprinkler solenoid to turn on or off based on specific conditions such as time, including sunrise and sunset, humidity, and temperature. By implementing this method, you can achieve better results with low effort.

Must Haves

First, the IoT device must run when the internet is down, or when the network is down.
Second, there must be timer-based watering, which includes support for sunrise and sunset.
Third, you need the ability to manually toggle the watering state.
Fourth, there must be backup and restore settings.
And finally, there is a future goal: if sunrise or sunset occurs with a temperature restriction, then start watering, in order to prevent the freezing of plant roots.

Wants

First, the system should measure water time, humidity, and temperature. All off-the-shelf sprinklers do this with various features.
Second, you want local network control with no data submitted to a third party.
Third, you want graphing and the ability to analyze the data.
Fourth, you want watering runaway protection.
As for future goals, if humidity is at a certain percentage, skip watering today. Most over-the-shelf sprinklers do this.
Another goal is: if the weather forecast is rainy, skip watering and notify me when this occurs. Most off-the-shelf sprinklers can do this sort of thing.
You also might want to send notifications to any platform you want, such as AWS, Gmail, IFTTT, or Alexa.
And lastly, integrating a moisture sensor.

Hardware

At the lowest level, the IoT hardware should do the following:
It should toggle power on and off.
It should report the power state, humidity, and temperature.
It should not require an internet connection.
And it should allow you to customize power toggling based on special conditions.

There is a twenty-dollar USD over-the-shelf Sonoff hardware that does those three out of four things easily. The fourth point requires special programming. There are multiple options out there, but this fulfills the need of an over-the-shelf device that can be used with open-source software.

The image shown here depicts the Sonoff device. The easiest way to wire this up is using an existing extension cord. Cut into it, then follow the directions of the wiring Sonoff guide. Plug one into the twenty-four VAC power supply of an existing water sprinkler adapter.

However, I ended up using alternative hardware. This included an ESP8266 microcontroller, an OLED screen, a solid state relay, and a BME280 sensor for temperature, humidity, and pressure.

Software

The device can be flashed with open-source software called Tasmota. I prototyped with the Arduino IDE for a while, but Tasmota seems safer with the features needed. So I won't go into too much detail, but Tasmota solves the following problems:
It is locally network controlled.
It runs even when the internet is down.
It is highly customizable and battle-tested for years.
It has simple programming.
And there are multiple ways of interacting with it, including command line, MQTT, and a web UI.

There is a video available on how to flash and set up the device when you get it, as this is really common with this device.

Wiring and Module Parameters

The wiring connections are as follows:
D2, GPIO4, connects to I2C SDA.
D1, GPIO5, connects to I2C SCL.
D5, GPIO14, connects to Relay 1.
D0, GPIO16, connects to Counter 1.

Setup Time

Using the Tasmota Web Console Command line, here are the steps:
First, type PowerOnState 0. This means if there is a power outage or reset, the default power state will be off.
Second, set your time zone using standard GMT offset by typing Timezone -8. This does not account for daylight savings time; you can find more info in the documentation link on the blog post.
Third, test your time by typing time.
Fourth, update your location. You can use a website like latlong.net to find your coordinates. Run Latitude 0.0000 for latitude, replacing 0.0000 with the proper value. Then run Longitude 0.0000 for longitude, again replacing 0.0000 with the proper value.
Finally, run STATUS 7 to see sunrise and sunset with local time. Ensure it is the actual time.

Setup Timers

Using the Tasmota Timer Web UI, four timers will be created. Two timers for starting and stopping the watering at sunrise, and two timers for the same at sunset.

The timer configurations are as follows:
Timer1 is enabled, Mode 1, Time 00:00, Window 0, Days all seven days, Repeat 1, Output 1, Action 1.
Timer2 is enabled, Mode 1, Time 00:20, Window 0, Days all seven days, Repeat 1, Output 1, Action 0.
Timer3 is enabled, Mode 2, Time 00:00, Window 0, Days all seven days, Repeat 1, Output 1, Action 1.
Timer4 is enabled, Mode 2, Time 00:20, Window 0, Days all seven days, Repeat 1, Output 1, Action 0.

Then, type Timers on.

It is also adjustable in the Web UI. The image here shows the Sunrise Starting Timer.

To stop the watering, Timer 3 has a twenty-minute offset. This allows for twenty minutes of watering. Do the same for Timer 4 for sunset. Per the Tasmota docs: When Mode 1 or Mode 2 is used, Latitude and Longitude become available. In that case, the Time value is always used as an offset, so make sure to set it to 00:00 if no offset is wanted.

The next image shows the Sunrise Stopping Timer.

Using the Tasmota Web Console Command line, type in the timers to verify: Timer1, Timer2, Timer3, and Timer4.

Runaway Protection

There are three methods for runaway protection:
First, Timer2 and Timer4 should stop the sprinkler.
Second, the system never runs for more than 1320 seconds, which is twenty-two minutes, when the power state is on. To do this, use the following rule: Rule1 on power1#state=1 do backlog RuleTimer5 1320; counter1 +1 endon on Rules#Timer=5 do power1 off endon. Then enable it with Rule1 1.
Third, use NodeRed to trigger a power state off if receiving MQTT messages. This could be adapted to turn off timers when the weather is expected to rain.
You can also monitor on and off times using Grafana. I have prototyped using alerts with a Telegram integration in case all else fails.

Extending to other Platforms (Optional)

Using a simple flow in NodeRed, I have added support for a UI and Alexa control with tracking usage via Grafana. Using the MQTT message tele/tasmota_YOURDEVICE/SENSOR, the data is formatted and sent to a local Grafana instance on a Raspberry Pi. You can find the link for installing Grafana on a Raspberry Pi on the blog post.

Setup Rules (Optional)

Here are two more timers, or rules, that are wanted:
First, before sunrise, check the temperature. If it is too cold, send an MQTT message and disable the start timers.
Second, before sunrise, check the humidity. If it is raining, then disable the timer.
