Top Five NuGet Packages

I use these NuGet projects consistently in most projects I'm building. I'll share examples in the details for usage.

FluentValidation

You can install this package using the command Install-Package FluentValidation.

Doing user input validation can be tedious. This makes it less so.

Here is an example of how it works. You define a validator class that inherits from AbstractValidator, passing in the type you want to validate, such as a Customer object. Inside the constructor, you set up rules. For instance, you might say the Surname should not be empty. The Forename should not be empty, with a custom message saying "Please specify a first name." The Discount should not equal zero, but only when the customer has a discount. The Address length should be between twenty and two hundred fifty characters. And the Postcode must pass a custom validation method called BeAValidPostcode, with a message if it fails. That validation method contains your custom logic for checking postcodes.

To use it, you create an instance of the Customer, then an instance of the CustomerValidator. You call the Validate method on the validator, passing in the customer. This returns a ValidationResult object. You can check the IsValid property to see if the validation passed, or access the Errors list to see what went wrong.

You can find the link to the GitHub reference on the blog post.

Microsoft.Extensions.DependencyInjection

Install this package using the command Install-Package Microsoft.Extensions.DependencyInjection.

Dependency injection, also known as inversion of control, is a popular pattern. Lately, I've stuck with the Microsoft version as it integrates nicely with ASP.NET and is fairly simple. Unit testing becomes easier since the interfaces can be mocked.

In this example, a service collection is created using interfaces with corresponding concrete implementations. Then one service is used within a scope.

First, you create a new ServiceCollection. Next, you add a singleton service for an interface IFoo, providing a concrete Foo implementation. Then, you add a scoped service for an interface IBar, mapping it to the concrete Bar class. After that, you build the service provider from the collection.

To use it, you create a scope using the provider. Within that scope, you get an instance of the IBar service. This ensures the service lifetime is managed correctly.

You can find the link to the usage reference on the blog post.

Easy.Common

Install this package using the command Install-Package Easy.Common.

Have you ever tried counting lines of a very large file? If not, it can get complicated quick.

This package provides a simple extension method to help. You create a FileInfo object pointing to your large file, such as a CSV file. Then, you open a read stream on that file. Finally, you call the CountLines method on the stream, which returns the total number of lines.

There are other useful extensions in this package as well. You can find the link to the GitHub reference on the blog post.

Xunit, Moq, AutoMoq

You can install these packages using the commands Install-Package xunit, Install-Package Moq, and Install-Package AutoFixture.AutoMoq.

Here is an example of mocking in action.

We have a test method decorated with the Fact attribute, named Can_get_message_from_cache. Inside the test, we first arrange the data. We use a fixture to create an expected value of type unsigned integer. Then, we set up a mock object to expect a call to the Get method with that expected value. When that call happens, it returns a Message object built using the fixture, which includes adding a header with the key UID and the value of the expected integer.

Next, we act by calling the GetMessageAsync method on the provider, passing in the expected value.

Finally, we assert that the result is correct. We check that the UID header in the returned message parses to the same expected integer value.

You can find the links to the Xunit getting started guide and the Moq tutorial on the blog post.

Polly

Install this package using the command Install-Package Polly.

Need retry logic? Exponential back off? This project solves the problems easily with tons of extensions.

Here is an example. You create a retry policy that handles a specific exception type, called TransientException. The policy is configured to wait and retry three times, sleeping for one second between each attempt. Then, you execute a block of code within that policy. In this case, the code logs the attempt number and throws a TransientException. The policy will catch the exception and retry automatically according to its configuration.

You can find the link to the GitHub reference on the blog post.

Honorable Mentions

There are several other packages worth noting.

First, LambdaSharp helps you deploy to AWS Lambda and resources easier and safer.

Second, PubSub implements the publish and subscribe pattern.

Third, M2MqttDotnetCore is an MQTT client.

Fourth, SynthesizerAudio allows you to synthesize text to speech.

Fifth, Microsoft.OpenApi is like Swagger but better.

Sixth, Microsoft.IdentityModel.Tokens and System.IdentityModel.Tokens.Jwt are for working with JWTs. They are not the easiest to read but are essential for understanding how to use them.

Seventh, Refit is a safe type REST library.

Eighth, S22.Imap.Core is for interacting with email via IMAP. The library is simple and easy to use.

Ninth, Microsoft.Extensions.Caching.Memory is for caching safely.

Tenth, EmbedIO is a cross-platform HTTP and WebSocket server.

Eleventh, Watson is a cross-platform HTTP server that has attribute routing as well.
