Automate Cleaning of Air Conditioner Condensate Drain Line

Ever have your central AC unit turn off all of a sudden? If you’re a homeowner, this is a common source of annoyance. It’s the most common reason why service is called for. The air conditioner condensate drain line is blocked. The solution is to clean the AC condensate drain line every month with a quarter cup of vinegar, then flush with water once a month. You have to remember to do this, though, and it’s easily forgotten. It should be done more frequently to prevent any mold or bacteria buildup. This post is a how-to guide to configure an off-the-shelf smart Wi-Fi dual water pump to flush out this line on a regular basis.

Here is the plan. First, flash the device with Tasmota firmware. Second, set up a water reservoir. Third, route the water line to the AC condensate drain line opening, which is usually near the start of the heating or fan unit. And finally, add rules to manage water pump timing and notifications.

Now, let’s look at the water pump preparation. I ordered this Wi-Fi dual water pump from Amazon, and you can find the link on the blog post. Then I flashed it using the guide available at blakadder.com, and you can find that link as well on the blog post. The only tricky part was soldering wires to those tiny connections shown in the guide’s image. I followed the traces for two of them and soldered onto those instead, giving me more room to work with.

I flashed it with Tasmotizer, which you can find on the GitHub releases page. Send the Wi-Fi configuration, then grab the IP address. Make sure to set the template defined in the guide at this point, by going to Configuration, Configure Other. The template is a JSON object with the name WD-01ADE, GPIO values of 32, 0, 0, 0, 0, 225, 33, 0, 224, 320, 0, 0, 0, 0, a flag of 0, and a base of 18.

In order to monitor the pump activations, a counter was added. This is useful to keep track of the amount of water or vinegar mixture left in the reservoir. This needs to be done after the template was set by going to Configuration, Configure Template, and setting GPIO 16 to Counter, with the counter drop-down selected as 1. The value is stored even after a reboot.

Next, we set up a water reservoir. Take a vinegar plastic bottle and poke a hole in it about the size of the fitting. After that, hot glue in one of the fittings that came with the water pump supplies. Be sure to poke a hole in the top so air can enter the bottle. There is an image here showing the plastic bottle with the fitting hot-glued in place.

Now, let’s configure the Tasmota rules. Add this rule when pump one has started. It will schedule to turn off in thirty seconds, which should disperse about one cup of vinegar and water mixture. It increments one to Counter 1. The rule reads: Rule 1, on power 1 state equals 1, do backlog RuleTimer2 30, counter1 plus 1, endon, on Rules Timer equals 2, do power 1 off, endon. Then type Rule1 on.

Turn on pump one every day at 4 a.m. within a fifteen-minute window. The command is Timer1 with Enable 1, Time 4:00, Window 15, Days SMTWTFS, Repeat 1, Output 1, Action 1. Then type Timers on.

Set the timezone of your location. For example, Timezone negative 8.

Lastly, if the device restarts, ensure the water pump relays are off by default. This is not the default with Tasmota firmware. Use the command PowerOnState 0.

All of these commands together are also available in a Gist, which you can find the link for on the blog post.

Final results for the web UI are shown in a screenshot of the Tasmota interface. There is also an MQTT Sensor topic that looks like this: tele/tasmota underscore XXXXX/SENSOR, with a time stamp and a counter value.

For future improvements, based on the counter, we could send an alert to refill the reservoir tank.

And finally, here are the resources mentioned. You can find the links on the blog post for the Tasmota device template, a guide on unclogging AC condensate lines, the Tasmotizer GitHub releases, the Tasmota Device Manager documentation, and the PowerOnState documentation.
