Building a Bluetooth proximity detector using a Raspberry Pi is a surprisingly simple project. All you need is the Raspberry Pi itself, a USB Bluetooth adapter, and a phone with Bluetooth turned on. This guide will walk you through detecting a known Bluetooth device within range of the Pi, without needing to pair the two devices together.

First, let's take a look at the hardware we are working with. The image shows the Raspberry Pi 2 Model B, a small, bare circuit board with its various ports and components clearly visible, which will be the brain of our proximity detector.

Now, onto the actual steps.

Step one is ensuring that your USB Bluetooth adapter is working on the Raspberry Pi. To do this, you need to install the necessary software. You would run the command to install bluetooth using apt-get. Once that is installed, you want to check the status of your Bluetooth connection. You can do this by running the hcitool dev command. If everything is set up correctly, it should return the device's Bluetooth address, looking something like "Devices: hci0 00:00:00:00:00:17".

Step two is getting the MAC address of the device you want to detect. If you are using an Android phone, for example running version 6.0.1, you can find this by going to Settings, then System, then About device, then Status, and finally looking for the Bluetooth address. Write this down, as we will need it for the next step.

Step three is testing the connection to your client device. To do this, you will use the l2ping command. Specifically, you would type "sudo l2ping -c 2" followed by the MAC address of your phone. So, if your phone's address is 00:11:22:AA:BB:CC, you would enter that exact string. When you run this, you should see a result indicating the ping response. For instance, you might see something like "Ping: 00:11:22:AA:BB:CC from 00:00:00:00:00:17" with data size details, followed by two responses showing the bytes received and the time taken, such as "0 bytes from 00:11:22:AA:BB:CC id 0 time 7.32ms" and "0 bytes from 00:11:22:AA:BB:CC id 1 time 22.46ms". This confirms that your Pi can see and communicate with your phone.
