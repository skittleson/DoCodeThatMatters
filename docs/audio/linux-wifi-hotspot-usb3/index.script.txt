This is a blog post titled Linux Wifi Hotspot with PiHole. The post was originally published on September 29, 2022, and modified on October 11, 2022. The description says to build a wifi hotspot using hostapd via the create_ap project and Pi-hole. There is an image on the post that depicts the Pi-hole logo, which looks like a vortex or funnel shape.

A quick note to start: this post is still a work in progress, so come back for updates.

Here is the TLDR section. You will need an Alfa AC1900 WiFi Adapter. It is a 1900 Mbps 802.11ac Long-Range Dual Band USB 3.0 Wi-Fi Network Adapter with four 5 dBi external dual-band antennas. You can find the link to this specific adapter on the blog post.

The script used to start the setup is named onstart.sh. Instead of reading the raw code block verbatim, here is what the script does: It is a bash script that first disables power saving on the wlan0 interface. Then, it sets the transmission power on wlan0 to a fixed value of 4500. Next, it writes the value 1 to the system parameter for the 8814au module to switch USB modes. Finally, it runs the create_ap command as a daemon. The command logs to a file at /home/support/create_ap.log, disables dnsmasq, sets the DHCP DNS to 192.168.12.1, sets the country to the US, and enables IEEE 802.11n and 802.11ac standards. It uses the 5 GHz frequency band and specifies various high-throughput capabilities like LDPC, HT40+, SHORT-GI, and MAX-AMSDU. It also specifies VHT capabilities like MAX-MPDU, RXLDPC, SHORT-GI-80, and beamforming options. The script creates a network named TestNetwork with the password testing123, using wlan0 for the wireless interface and eth0 for the uplink.

Next, you need to disable IPv6. To do this, edit the sysctl configuration file using the command sudo nano /etc/sysctl.conf. You need to add three lines to the file: the first disables IPv6 for all network interfaces, the second disables it for the default interface, and the third disables it for the loopback interface.

Then, install and configure Pi-hole. You can do this by running the Pi-hole installation script from the command line. The command involves using curl to fetch the install script from install.pi-hole.net, setting the environment variable PIHOLE_SKIP_OS_CHECK to true, and running it with sudo and the -E flag. After installation, set up the DHCP server to use the 192.168.12.1 range.
